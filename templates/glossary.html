Metrics Guide

Spacing Scale
What we're checking
Whether spacing follows a consistent mathematical pattern (powers of 2, multiples of base unit, etc.)
Why it matters
Consistent spacing creates visual rhythm and makes designs feel cohesive. Sites like Stripe, Airbnb, and Apple use mathematical spacing scales for this reason.
Base unit explained
The smallest spacing increment. All other values are multiples of this base (4px → 8px, 16px, 32px).
Confidence scoring
Factor Points
Base unit found +40pts
Pattern consistency +30pts
Unique multiples +20pts
No outliers +10pts
Score ranges
95-100%: Perfect geometric scale with 30+ instances
80-94%: Strong pattern with minor outliers
60-79%: Noticeable pattern but inconsistent
40-59%: Weak pattern, many outliers
Responsive Breakpoints
What we're checking
Screen width breakpoints where layout changes for different devices (mobile, tablet, desktop)
Why it matters
Ensures the site looks good on mobile, tablet, and desktop. 60%+ of traffic is mobile nowadays.
Common breakpoints
768px: Tablet
1024px: Desktop (small)
1440px: Desktop (wide)
Frameworks: Bootstrap (576px, 768px, 992px, 1200px), Tailwind (640px, 768px, 1024px, 1280px)
Shadow System
What we're checking
Consistent box-shadow values used for depth and elevation hierarchy
Why it matters
Creates visual depth and helps users understand element layering (what's in front, what's behind)
Example elevation system
Level 1: 0 1px 3px rgba(0,0,0,0.12) - Cards, tiles
Level 2: 0 4px 6px rgba(0,0,0,0.16) - Dropdowns, tooltips
Level 3: 0 10px 20px rgba(0,0,0,0.2) - Modals, overlays
Similar to: Material Design elevation, Apple iOS shadows
Z-Index Stacking
What we're checking
CSS z-index values that control which elements appear on top of others
Why it matters
Proper stacking prevents visual bugs (modals appearing behind content, dropdowns hidden, etc.)
Why values above 500 are hidden
React portals use z-index: 9999, tooltips use 999999. These aren't part of the design system - they're framework noise. We only show meaningful layers (1-500).
Typical stacking hierarchy
1-10: Base content, images
100: Dropdowns, popovers
200: Sticky headers
300: Modals, dialogs
400: Notifications, toasts
Accessibility Score
What we're checking
Overall accessibility quality based on WCAG (Web Content Accessibility Guidelines)
Why it matters
Makes the site usable for people with disabilities (screen readers, keyboard navigation, color blindness) and improves SEO
Score breakdown
Factor Points
Semantic HTML (header, nav, main) +25pts
ARIA labels present +15pts
Alt text on images +15pts
Keyboard navigation working +15pts
Good color contrast (4.5:1) +15pts
Skip navigation link +10pts
Score ranges
90-100%: Excellent - WCAG AAA compliant
75-89%: Good - WCAG AA compliant
60-74%: Fair - Some issues
40-59%: Poor - Many issues
SEO Score
What we're checking
Search engine optimization quality (meta tags, structure, performance)
Why it matters
Better SEO means more organic traffic from Google, Bing, etc. Good SEO = more visitors without paid ads.
Key elements
Title tag: Unique, 50-60 characters
Meta description: 155-160 characters
Open Graph tags: Social media sharing
Schema.org markup: Rich snippets in search
Mobile responsive: Required
Load time: Under 3 seconds
Security Score
What we're checking
Security best practices (HTTPS, CSP, CORS, security headers)
Why it matters
Protects users from XSS attacks, CSRF attacks, clickjacking, and other web vulnerabilities
Key security headers
HTTPS: Encrypted connection (locks in browser)
CSP: Content Security Policy (prevents XSS)
X-Frame-Options: Prevents clickjacking
X-Content-Type-Options: Prevents MIME sniffing
Strict-Transport-Security: Forces HTTPS
CSS Tricks
What we're checking
Modern CSS features and techniques used on the site
Why it matters
Shows technical sophistication and progressive enhancement. Modern CSS = better performance and maintainability.
Categories we detect
CSS Variables: Custom properties (--color-primary, --spacing-base)
Flexbox: Modern layout system
Grid: Advanced 2D layout (grid-template-columns)
Custom Properties: @supports, clamp(), calc()
Transforms: scale(), rotate(), translate()
Animations: @keyframes, transition
Filters: blur(), brightness(), contrast()
Blend Modes: mix-blend-mode
What "947 detected" means
This is the total count of modern CSS technique instances across the entire page. Click "View All" to see them categorized (e.g., "234 CSS Variables", "187 Flexbox containers").
